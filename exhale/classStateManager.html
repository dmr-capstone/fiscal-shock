class StateManager¶

Represents a play session. Consider this the player’s “save data.” Save games could be implemented by serializing this to JSON (may require a separate backing data class with no methods).

Property

property StateManager::cashOnHand

Current cash on hand. Cash on hand is the player’s health and ammo.

property StateManager::nextID

Next available loan ID.

property StateManager::timesEntered

Number of times the player has entered the dungeon.

property StateManager::totalFloorsVisited

Number of floors the player has visited inside the dungeon. A low ratio of timesEntered : totalFloorsVisited implies a more successful player, as they were able to go deeper in the dungeon.

property StateManager::currentFloor

Current dungeon floor the player is on. Used for difficulty scaling.

property StateManager::scoreChangeFactor

Used for calculating credit score.

property StateManager::creditScore

Current credit score of the player.

property StateManager::paymentStreak

Number of days in a row that the player has made regular payments on all of their active loans.

property StateManager::cashOnEntrance

Amount of money the player had before entering the dungeon. Used to calculate income.

property StateManager::selectedDungeon

Currently selected dungeon type. Used by the dungeon generator.

Public Static Functions

static IEnumerator StateManager.makePauseAvailableAgain()

Hitting “esc” to exit GUIs sometimes hits the pause code too, depending on order of execution. Bad things happen when the pause menu has a different order of execution. So, this is the nicest way to avoid bringing up the pause menu when someone exits a shop via keyboard.

static void StateManager.startNewDay()

Calls creditor functions to accumulate interest and gets income earned from the last dungeon dive.

Return

static void StateManager.calcCreditScore()

Calculates the player’s credit score. Used to apply bonuses/penalties to interest rates and maximum loans

static void StateManager.resetToDefaultState()

Resets the StateManager to the default state. Use this when the player’s session needs to be reset. The StateManger is set up for a “fresh” playthrough.

static void StateManager.destroyAllSingletons()

Destroy all singletons the state manager is keeping track of.

Whether the player is currently in the story tutorial. The player should not spend any money while in the tutorial.
